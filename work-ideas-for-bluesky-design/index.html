Work Ideas

A list of ideas for ways we could improve things.

Code Style guide

e.g Only camel case or kebab case - but not both - use cases etc

Docs for each site

README on repo, or Gist, or page on dev site, updated with any useful stuff e.g. img correct px dimensions for banners, pages, as set up by build designer

Img / asset database

One source of truth for all the latest logos / images / fonts for brands to use across sites. Save duplication of jobs sourcing these.

Node Sharp - on server.

Set default image sizes for all of our sites for md, lg, default. Then can upload folders of images and it'll give back cropped image automatically.

Tags on our css icons

Problem: it takes too long to search the icons for what you need Have a master copy - on Github? With the tags etc used for reference.

Modular Pages

Modular pages - components - built up of dirs / sub-dirs Reduce amount of code in any 1 file - easier to navigate. Easier to move to includes when there is a lot of repetition. Less chance of Designers needing same file at same time
